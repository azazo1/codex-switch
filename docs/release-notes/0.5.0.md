# Codex Switch v0.5.0

本版本扩展了 Chat Completions 兼容上游的工具调用转换, 增加可配置的上游错误重试和 GUI 构建版本展示, 同时改进了活跃连接输出观察与隔离调试.

## Highlights

- Responses 请求现在可以更完整地转发给只支持 Chat Completions 的 OpenAI 兼容上游, 包括文本和图片消息, function tool, custom tool, namespace tool, tool search, tool output, reasoning 和 usage.
- 每个上游可以选择关闭, 仅临时错误或全部上游错误的重试策略. 重写后的响应会进入 Codex 自身的有限重试链路.
- GUI 底部状态栏显示构建版本. 精确 tag 显示 tag, 非 tag 构建显示最近 tag 加 commit, 脏工作区构建显示 `tag^commit`.

## Chat Completions Compatibility

- Chat Completions 上游会接收标准 `messages` 和 `tools`, 并可将流式或 JSON 响应恢复为 Responses 事件和输出项.
- custom tool 会转换为接收 `input` 字符串的 function tool. namespace tool 会展开为受长度限制的稳定名称, 回程时恢复原始 namespace 和名称.
- `developer` 消息会降级为 `system`, 以兼容更多 Chat Completions 服务. 私有工具类型, 私有消息结构, 音频和文件输入仍不在自动转换范围内.
- 原生 Responses 上游仍适合依赖完整 Responses 语义或 provider 私有扩展的工作流.

## Upstream Retry

- 新增每上游保存的错误重试策略, 默认关闭.
- `仅临时错误` 会处理过载, 降速和普通限流. `全部上游错误` 还会处理上下文超限, 额度, 策略和无效请求等上游分类不准确的情况.
- SQLite 会自动执行 schema migration 12, 为现有上游增加默认关闭的重试策略. 不需要手动迁移数据库.

## Debugging And Interface

- `just debug` 使用隔离数据库启动桌面应用, 可在显式开启后记录完整代理 body. Authorization 和保存的 API Key 不会写入该日志.
- 活跃连接实时输出窗口关闭了截断文本的自动 hover, 保留稳定的可滚动完整输出窗口.
- GitHub Actions 会保留 Git 历史, 使非 tag 构建能够解析最近的发布 tag.

## Upgrade Notes

- 本版本不需要清除现有数据库或重新导入上游.
- 升级后保持原有上游行为. 只有手动启用错误重试策略后, 上游错误才会转换为可重试响应.
