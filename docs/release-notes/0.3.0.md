本版本改进 OAuth 账号导入和活动连接观察体验, 并修正仪表盘中已删除上游的残留统计.

## Highlights

- Codex OAuth 账号现在可以只通过 access token 导入, 不再强制要求 refresh token.
- 活动连接可以在悬浮窗口中查看多行流式输出, 并明确区分请求中, 等待输出和非流式响应.
- 仪表盘, 上游列表和活动连接表支持在窗口内滚动, 数据较多时仍可完整操作.

## OAuth

- 导入缺少 refresh token 的 `auth.json` 时, 应用会保存有效 access token 并标记为 `仅 access token`.
- 更新已有账号且新文件不含 refresh token 时, 会保留原有 refresh token, 避免账号失去自动刷新能力.
- 仅 access token 的账号可以使用到 token 临近过期, 到期后会返回明确的重新导入提示.

## Active Connections

- 流式输出悬浮窗口保留最近 `10` 个逻辑行和 `768` 个 Unicode 字符, 支持滚动查看并自动跟随最新内容.
- 活动连接从请求发出开始显示 `请求中`, 收到响应头后再切换为流式或非流式状态.
- 活动连接表限制在可用窗口高度内, 大量并发请求不会把后续界面内容挤出视口.

## Dashboard And Upstreams

- 仪表盘和上游列表支持纵向滚动, 上游列表移除代理展示列, 代理配置能力保持不变.
- 按上游统计只展示当前仍存在的上游. 删除上游后, 历史请求日志和总计数据会保留, 对应统计行不再显示.

## Compatibility

- Codex OAuth 请求头版本更新为 `0.144.2`.
- 本版本不需要手动迁移数据库. 替换程序后会继续使用现有 SQLite 数据.
