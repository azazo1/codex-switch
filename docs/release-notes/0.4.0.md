本版本改进活跃连接的流式输出观察体验, 增加可控滚动, 实时输出速度和关闭状态提示, 同时让连接表格适配默认窗口宽度.

## Highlights

- 实时输出窗口可以限制最快滚动速度, 默认保持不限速, 并支持设置关闭连接滚到末尾后的保留时间.
- 活跃连接新增输出速度列, 使用模型对应的 tiktoken 编码估算最近 2 秒的可见文本速度, 主列显示 `tps`, 悬浮显示字符速度.
- 已关闭连接会使用弱化颜色继续展示尚未滚完的内容, 到达末尾并完成保留计时后再移除.
- 活跃连接表采用紧凑固定列宽, 默认窗口内不再产生额外横向溢出.

## Active Connections

- 主输出缓冲扩展为最近 `16384` 个 Unicode 字符, 并使用绝对字符位置维护独立滚动游标.
- 启用限速后, 每条连接按配置的字符速度推进. 输出超过缓冲上限时会从当前最早可用位置继续.
- 连接关闭后会冻结持续时间和输出速度, 禁用终止按钮, 并从页签活跃数量中立即扣除.
- 默认末尾保留时间为 `2` 秒, 可调整为 `0` 到 `300` 秒. 所有显示设置仅在当前运行期有效.

## Output Speed

- 输出速度使用最近 2 秒实际收到的 SSE 文本 delta 计算, 不受窗口滚动速度影响.
- 已知模型使用对应的 tiktoken 编码, `openai/` 前缀会自动移除, 未知模型回退到 `o200k_base`.
- 主列始终使用 `~` 标识估算 `tps`, 避免将中转模型分词差异或不可见 reasoning token 表示为精确值.
- 连接停止输出超过统计窗口后速度会降为 `0`, 关闭后保留最后一次速度供观察.

## Interface

- 上游, 模型, 推理强度和 endpoint 使用紧凑固定列宽, 过长内容可以通过悬浮查看完整值.
- 输出速度, 已持续, 开始时间和操作列使用稳定宽度, 实时更新不会改变表格布局.
- 实时输出窗口根据剩余空间在 `120` 到 `480` 像素之间调整.

## Upgrade Notes

- 本版本不需要手动迁移数据库.
- 活跃连接显示设置不会写入 SQLite, 应用重启后恢复默认值.
