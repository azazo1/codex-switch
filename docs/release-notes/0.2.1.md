本版本累计整理 `v0.1.0` 之后的全部用户可见变更, 不包含 `v0.1.0` 及更早内容.

## Highlights

- 支持同时管理多个 Codex OAuth 设备授权任务, 每个任务独立展示授权码, 轮询状态, 重试和过期信息.
- 支持一次导入多个 Codex CLI `auth.json` 文件, 并按文件报告成功, 更新, 跳过或失败结果.
- 为桌面应用, 本地代理, 上游, OAuth, 调度, 缓存保持, 余额提醒, 活跃连接, 日志和存储补全使用文档.

## Features

- 设备授权任务支持自动轮询, 退避重试, 到期处理, 手动重试和取消, 多个登录流程可以并行进行.
- 批量导入会验证凭据结构, 账号身份和 token 有效期, 单个文件失败不会阻止其他有效账号导入.
- 已存在的 OAuth 账号会在事务中更新凭据, 同时保留上游名称, 启用状态, 调度组, 权重和代理设置.
- 新导入账号会建立独立上游, 界面可以刷新列表并继续管理对应额度和路由配置.
- tracing 日志不会记录 access token, refresh token, id token 或完整凭据内容.

## Documentation

- 新增根目录快速开始, 下载方式, 安全边界和完整文档索引.
- 新增应用, 代理, 上游, OAuth, 余额提醒, 活跃连接, 日志, 存储和发布指南.
- 补充调度组嵌套, 缓存保持会话, SQLite 备份和本地访问 key 的实际行为说明.

## Breaking Changes

- Release 资产名现在包含版本和标准架构名, 自动下载脚本需要匹配 `codex-switch-<version>-<platform>-<arch>.<ext>`.

## Packaging

- 为 Linux, Windows 和 macOS 提供 `x86_64` 与 `aarch64` 共 6 个构建产物.
- 每个平台在归档前检查最终二进制格式, Release 在发布前验证 6 个归档全部存在.
- Release 附带 `SHA256SUMS`, 用于校验所有下载归档.

## Upgrade Notes

- 本版本不需要手动迁移数据库. 替换程序后会继续使用现有 SQLite 数据.
- 导入与现有账号匹配的 OAuth 凭据时, 原有路由和代理设置会保留.
