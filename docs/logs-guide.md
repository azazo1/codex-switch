# 日志使用指南

Codex Switch 有两类日志:

- `日志` 页面保存代理请求记录, 用于查看用量, 状态和性能.
- tracing 运行日志记录程序阶段和错误, 用于诊断应用本身.

清理请求日志不会删除 tracing 运行日志.

## 请求日志字段

| 字段 | 说明 |
| --- | --- |
| 上游 | 实际使用的上游, 悬浮可查看状态和错误 |
| 模型 | 客户端模型名, 请求失败时附带错误标记 |
| 推理强度 | 请求中的 reasoning effort |
| TOKEN | 输入, 缓存输入, 缓存写入, 输出和总 token |
| 费用 | 写入日志时根据价格缓存计算的 USD 估算 |
| 首 TOKEN | 流式响应收到首段数据前的时间 |
| 耗时 | 请求从进入代理到结束的总时间 |
| 时间 | 请求完成时间, 按本地时区显示 |

悬浮上游或模型可以查看 endpoint, status 和错误文本. 点击 token 数字可以切换简写和精确数字.

请求日志不保存 prompt 或完整响应正文. 它会保存模型名, 上游, endpoint, 状态码, token, 费用估算, 耗时和错误文本等元数据.

一行日志代表一次上游尝试或一次内部缓存保持请求, 不一定代表一个独立客户端请求. 失败切换可能为一个客户端请求产生多行日志, `/internal/cache_keepalive` 也会计入仪表盘请求数和用量汇总.

## 分页

每页可以显示 `10`, `20`, `50` 或 `100` 条记录. 切换每页数量会回到第一页. 顶部会显示当前结果范围和筛选后的总条数.

## 筛选

点击 `筛选日志` 可以组合以下条件:

- 模型, 支持 `*` 和 `?` 通配符.
- 上游, 推理强度和 endpoint.
- 状态码类别或自定义范围.
- 日志完成时间范围, 界面使用开始和结束两个边界输入.
- USD 费用范围.
- 总耗时和首 token 延迟范围.
- 输入, 输出, 缓存输入, 缓存写入和总 token 范围.

Token 范围支持 `K`, `M`, `B` 和小数, 例如 `1.5M`. 最小值大于最大值或输入无法解析时, 状态栏会显示筛选错误.

部分下拉候选从当前页记录生成. 较早记录的模型, endpoint 或推理强度可能不在候选中, 其中模型字段可以直接输入通配符.

## 清理

点击 `清理日志` 后可以选择:

| 策略 | 实际行为 |
| --- | --- |
| 保留一天 | 删除早于当前时间 1 天的记录 |
| 保留一周 | 删除早于当前时间 7 天的记录 |
| 保留一个月 | 删除早于当前时间 30 天的记录 |
| 保留一年 | 删除早于当前时间 365 天的记录 |
| 保留指定条数 | 只保留最新 N 条 |
| 只清理失败请求 | 删除 `status >= 400` 的记录 |

清理后会从剩余请求重新构建用量汇总. 因此仪表盘的历史请求数, token 和估算费用也会下降. 当前没有请求日志导出或自动保留周期.

删除记录不会立即缩小 SQLite 主文件. 空闲页面可以在仪表盘数据库区域观察.

## 时间和费用边界

日志列表时间使用本地时区. 仪表盘的 `今日` 汇总当前按 UTC 日期计算, 在本地午夜附近可能与日志列表的自然日范围不同.

单条日志费用使用请求完成时的价格缓存. 后续刷新模型价格不会回写旧日志费用, 但仪表盘汇总会按当前缓存重新估算, 两处数值可能不同.

## Tracing 运行日志

Windows GUI 没有控制台, tracing 默认以 `info` 级别追加到数据目录中的 `codex-switch.log`. 当前没有文件轮转.

Linux 和 macOS 默认把 tracing 写到标准错误. 从桌面图标启动时, 不保证存在可回看的日志文件. 从终端运行可以直接查看输出.

可以在启动进程前设置 `RUST_LOG` 调整日志级别. 数据目录位置见[存储与备份指南](storage-guide.md).
