# 活跃连接指南

`活跃连接` 页面显示当前正在向上游发送或读取响应的请求, 以及已经关闭但仍在补完输出滚动或等待移除的请求. 页签名称中的三位数字只统计实际仍活跃的连接, 最大显示为 `999`.

## 页面字段

| 字段 | 说明 |
| --- | --- |
| 上游 | 当前实际处理请求的上游 |
| 模型 | 客户端请求中的模型名 |
| 推理强度 | 请求中的 reasoning effort, 不存在时显示 `-` |
| endpoint | 转发后的内部端点标识 |
| 实时输出窗口 | 流式响应的最近一段文本, 或非流式状态 |
| 实时输出速度 | 最近 2 秒实际收到的文本对应的估算 token 速度 |
| 已持续 | 从开始向上游请求到当前的持续时间 |
| 开始时间 | 本地时区的开始时间 |

请求完成或失败后会先变为弱化颜色, 终止按钮同时失效. 如果输出窗口尚未滚到末尾, 它会继续滚动, 到达末尾并经过设置的保留时间后才从页面消失. 历史结果应到[日志使用指南](logs-guide.md)中查看.

## 显示设置

页面顶部可以设置实时输出窗口的滚动行为.

- `限制最快滚动速度` 默认关闭, 此时输出窗口直接追随最新文本.
- 启用限制后, 可以在 `1` 到 `1000` 字符/秒之间设置最大滚动速度, 默认值为 `60` 字符/秒.
- `末尾保留时间` 控制已关闭连接滚到末尾后继续显示多久, 范围是 `0` 到 `300` 秒, 默认值为 `2` 秒.
- 控件修改后立即应用. 这些显示设置只在当前运行期有效, 不写入 SQLite.

## 实时输出窗口

流式请求会为页面单元格保留最近 `16384` 个 Unicode 字符. 换行会转换为空格, 页面根据当前宽度和滚动游标显示其中一段. 当输出速度长期超过滚动上限并占满缓冲时, 页面会跳过已经淘汰的最早内容, 从当前最早可用位置继续滚动.

悬浮窗口会原样拼接流式 delta, 最后一行随新 delta 增量显示. 窗口最多保留 `10` 个逻辑行和 `768` 个 Unicode 字符, 达到上限时优先整行移除最早内容, 只剩一行时才从行首逐字符移除. 悬浮窗口默认跟随最新输出, 并支持纵向滚动查看.

上游响应头尚未返回时显示 `请求中`. 确认为非流式响应后显示 `非流式`. 确认为流式响应但尚未收到文本时显示 `等待输出`.

这个尾部只保存在内存, 不会写入请求日志. 但是界面可能短暂显示模型输出中的敏感内容, 共享屏幕时应注意.

## 实时输出速度

实时输出速度根据代理实际收到的 SSE 文本 delta 计算, 不受输出窗口滚动速度影响. 主列显示形如 `~12.3 tps` 的估算值. 鼠标悬浮后会显示同一统计窗口内的字符速度.

应用使用与模型名匹配的 tiktoken 编码估算可见文本 token 数. `openai/` 前缀会先移除, 无法识别的模型回退到 `o200k_base`. 中转模型可能使用不同分词器, usage 也可能包含不可见的 reasoning token, 因此数值始终使用 `~` 表示估算.

连接停止输出超过 2 秒时, 活跃连接的速度会降为 `0`. 连接关闭后冻结最后一次速度, 在补滚和末尾保留阶段继续显示.

## 终止请求

点击 `终止` 会取消当前上游尝试. 按钮会进入 `终止中` 状态, 当前尝试结束后从列表移除.

连接实际关闭后, 按钮会显示 `已关闭` 并保持不可用. 即使关闭事件与点击发生竞态, 存储层也不会再向已关闭连接发送终止信号.

流式请求通常会中断客户端正在接收的响应. 非流式请求如果位于失败切换组, 并且网络失败切换已开启且仍有候选, 可能继续尝试备用上游, 因此不保证终止整个客户端请求.

终止可能写入失败请求日志. 它不会撤销上游已经完成的计算或费用, 也不会停止同一会话中的其他请求.

如果请求在点击前已经完成, 状态栏会显示该活跃请求已经结束.

## 可见范围

请求只有在完成调度并开始访问上游后才进入活跃列表. 路由解析阶段直接失败的请求可能不会出现. 缓存保持内部请求也不作为普通活跃连接展示, 应在日志中筛选 `/internal/cache_keepalive`.
