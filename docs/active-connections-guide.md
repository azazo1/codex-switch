# 活跃连接指南

`活跃连接` 页面显示当前正在向上游发送或读取响应的请求. 页签名称中的三位数字表示当前活跃数量, 最大显示为 `999`.

## 页面字段

| 字段 | 说明 |
| --- | --- |
| 上游 | 当前实际处理请求的上游 |
| 模型 | 客户端请求中的模型名 |
| 推理强度 | 请求中的 reasoning effort, 不存在时显示 `-` |
| endpoint | 转发后的内部端点标识 |
| 实时输出窗口 | 流式响应的最近一段文本, 或非流式状态 |
| 已持续 | 从开始向上游请求到当前的持续时间 |
| 开始时间 | 本地时区的开始时间 |

请求在完成或失败后会立即从页面消失. 历史结果应到[日志使用指南](logs-guide.md)中查看.

## 实时输出窗口

流式请求会为页面单元格保留最近约 `768` 字节的文本尾部. 换行会转换为空格, 页面只显示适合当前宽度的末尾部分. 悬浮窗口会原样拼接流式 delta, 最后一行随新 delta 增量显示. 窗口最多保留 `10` 个逻辑行和 `768` 个 Unicode 字符, 达到上限时优先整行移除最早内容, 只剩一行时才从行首逐字符移除. 悬浮窗口默认跟随最新输出, 并支持纵向滚动查看.

上游响应头尚未返回时显示 `等待响应`. 确认为非流式响应后显示 `非流式`. 确认为流式响应但尚未收到文本时显示 `等待输出`.

这个尾部只保存在内存, 不会写入请求日志. 但是界面可能短暂显示模型输出中的敏感内容, 共享屏幕时应注意.

## 终止请求

点击 `终止` 会取消当前上游尝试. 按钮会进入 `终止中` 状态, 当前尝试结束后从列表移除.

流式请求通常会中断客户端正在接收的响应. 非流式请求如果位于失败切换组, 并且网络失败切换已开启且仍有候选, 可能继续尝试备用上游, 因此不保证终止整个客户端请求.

终止可能写入失败请求日志. 它不会撤销上游已经完成的计算或费用, 也不会停止同一会话中的其他请求.

如果请求在点击前已经完成, 状态栏会显示该活跃请求已经结束.

## 可见范围

请求只有在完成调度并开始访问上游后才进入活跃列表. 路由解析阶段直接失败的请求可能不会出现. 缓存保持内部请求也不作为普通活跃连接展示, 应在日志中筛选 `/internal/cache_keepalive`.
