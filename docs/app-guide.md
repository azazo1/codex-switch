# 应用使用指南

桌面界面负责启动本地代理, 管理运行状态并提供所有功能入口. 应用进程退出后, 代理服务, 缓存保持和余额提醒都会停止.

## 首次启动

首次启动会自动完成这些初始化:

- 创建本地 SQLite 数据库.
- 生成格式为 `cs-<uuid>` 的本地访问 key.
- 保存默认监听地址 `127.0.0.1:15721`.
- 创建不可删除的 `Default` 失败切换调度组.

代理服务不会随桌面应用自动启动. 添加上游后, 需要在仪表盘点击 `启动`.

## 页面布局

顶部页签对应主要功能:

| 页签 | 用途 |
| --- | --- |
| 仪表盘 | 启停服务, 管理本地 key, 查看统计和数据库状态 |
| 上游 | 管理 API Key 上游, OAuth 账号, 余额和提醒 |
| 调度组 | 选择上游池并配置模型路由 |
| 缓存保持 | 查看当前运行时 prompt cache 会话 |
| 活跃连接 | 查看并终止正在转发的请求 |
| 日志 | 查看, 筛选和清理持久化请求记录 |

顶部 `刷新` 会从数据库重新读取主要视图数据. 窗口底部状态栏显示最近一次操作结果.

## 启动和停止代理

仪表盘中的 `监听地址` 必须是完整的 IP 和端口, 例如 `127.0.0.1:15721`. 修改地址后, 点击 `启动` 才会保存新值并开始监听.

启动失败的常见原因:

- 地址格式无效.
- 端口已被另一个进程占用.
- 同时启动了第二个 Codex Switch 实例.
- 当前用户无权监听指定地址或端口.

点击 `停止` 只停止代理监听, 不会退出桌面应用. 当前没有开机自启或后台系统服务功能.

## 本地访问 key

点击仪表盘显示的 key 可以复制. 客户端应发送:

```text
Authorization: Bearer <本地访问 key>
```

点击 `刷新 key` 后, 可以使用随机值或填写任意非空值. 新值保存后立即生效, 已经使用旧 key 的客户端会在后续请求中收到 `401`.

## 用量和费用

仪表盘显示总请求, 今日请求, token 用量和 USD 费用估算. 点击 token 数字可以在 `K/M/B` 简写与精确整数之间切换.

这里的请求数按持久化请求日志行统计, 更接近上游尝试次数, 不是去重后的客户端请求数. 失败切换会为多个候选分别记录日志, 缓存保持内部请求也进入同一统计, 所以请求数可能高于客户端实际发起次数.

价格数据来自 `https://models.dev/api.json`. 本地价格缓存为空时, 应用会在启动后异步获取一次. 已有缓存不会按时间自动刷新, 可以使用 `获取模型价格` 手动替换缓存.

费用只用于参考:

- 上游未返回完整 usage 时, token 统计可能不完整.
- 未匹配到价格的模型不会计入估算.
- 汇总费用按当前价格缓存重新计算, 单条日志保留请求写入时的估算.
- Relay 的实际账单, 折扣和模型别名可能与公开价格不同.

## 按上游统计

按上游表格显示请求数, 输入 token, 缓存输入, 输出, 总 token, 估算费用, 缓存保持状态和余额. API Key 上游可以在这里手动刷新余额.

余额和额度属于最近一次持久化快照, 不代表供应商页面上的实时值. 使用方式见[上游管理指南](upstream-guide.md), [OAuth 使用指南](oauth-guide.md)和[余额提醒指南](balance-alert-guide.md).

## 数据库状态

仪表盘底部显示 SQLite 实际路径, 主文件, WAL, SHM, 页面数和请求日志条数. `打开位置` 会调用系统文件管理器定位数据库.

复制或迁移数据库前应先从托盘退出应用, 具体步骤见[存储与备份指南](storage-guide.md).

## 系统托盘

系统托盘初始化成功后, 关闭主窗口只会隐藏窗口, 不会停止服务.

- 左键托盘图标重新打开主界面.
- 右键菜单可以打开主界面, 启动或停止服务, 以及退出应用.
- 托盘 `退出` 会先停止代理, 再结束进程.
- 如果托盘初始化失败, 关闭主窗口会正常退出应用.

macOS 隐藏窗口时也会隐藏 Dock 图标, 再次打开窗口时恢复. 应用没有单实例锁, 不建议同时运行多个实例.

## 网络边界

默认监听回环地址, 只允许本机连接. 如果改为 `0.0.0.0:<端口>` 或局域网地址, 需要自行处理防火墙和访问控制.

代理服务使用明文 HTTP, CORS 允许任意来源, 请求体没有应用层大小上限. 即使配置了本地 key, 也不应直接暴露到公网.
